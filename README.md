## Project Description

This assignment focuses on testing a Singlish to Sinhala transliteration system using automated test cases.

The objective is to identify failures in the system by comparing the actual output generated by the system with the expected output defined in the test cases.

## How It Works

* The script reads each test case from the Excel file.
* It sends the Singlish input to the web application.
* It captures the Sinhala output generated by the system.
* It compares the actual output with the expected output.
* The result is marked as PASS or FAIL.

---

## Test Case Summary

* Total Test Cases: 50
* Most test cases are designed to FAIL in order to identify weaknesses in the system.

---

## Notes

* Expected outputs are written according to realistic system behavior (Sinhala + English mix).
* Test cases are original and not copied from any provided examples.
* Only Input and Expected Output columns were manually filled.

---

## Conclusion

The automated testing approach successfully identified inconsistencies and limitations in the transliteration system. The majority of test cases resulted in FAIL, demonstrating areas where the system does not accurately convert Singlish into Sinhala.
